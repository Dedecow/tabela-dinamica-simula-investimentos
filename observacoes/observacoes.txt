Projeto Final: Simulador interativo em JavaScript

Objetivos gerais

Criar um simulador interativo

Pode ser um E-commerce, Cotador de produtos/serviços, Simulador de Home banking com transações, Conversor de moedas/criptomoedas, entre outros.

Objetivos específicos

Utilizar dados remotos (ou simular com JSON)

HTML interativo (gerado a partir do JS)

Uso das ferramentas mais importantes de JS, e bibliotecas externas

Deve ser 100% funcional e com a lógica de negócios do seu projeto escolhido (Exemplo: se cria um E-commerce, simula o processo completo de compra)

Deve ser entregue

Projeto HTML + CSS + JS funcional

Formato

Arquivo em formato .ZIP (NÃO em formato RAR)

Link para subpasta do GDrive

Link para repositório do Github (preferencial)

deve ter o nome “ProjetoFinal+Sobrenome”

Sugestões

Elimine console.log e derivados

Substitua alert, prompt, confirm por uma biblioteca JS

Se incluir formulários para completar,
pré-carregue dados nos campos

Critérios de avaliação
Funcionalidade
Simula um ou mais fluxos de trabalho em termos de entrada-processamento-saída e não são observados erros de cômputo.

Interatividade
Captura entradas usando inputs e eventos adequados. As saídas são coerentes em relação aos dados inseridos e são visualizadas no HTML de forma assíncrona.

Escalabilidade
Declara funções com parâmetros para definir instruções com uma tarefa específica. Define objetos com propriedades e métodos relevantes ao contexto. Utiliza arrays para agrupar valores e objetos de forma dinâmica. O percurso das coleções é ótimo.

Integridade
Define o código JavaScript em um arquivo .js, referenciando-o corretamente a partir do HTML. A informação estática em formato JSON é utilizada adequadamente, sendo carregada de forma assíncrona.

Legibilidade
Os nomes de variáveis, funções e objetos são significativos para o contexto, as instruções são escritas de forma legível e são usados comentários oportunos. O código fonte é organizado em termos de declaração e sequência.
